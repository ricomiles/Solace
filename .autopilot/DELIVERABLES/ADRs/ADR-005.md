# ADR-005: Auto-save via Task debounce over Combine

Status: Accepted
Date: 2026-05-08

## Context
Auto-save must debounce text changes at 1.5s. Two common Swift approaches: Combine's `debounce` operator on a `PassthroughSubject`, or structured concurrency with a `Task` that `Task.sleep`s and is cancelled on each keystroke.

## Decision
Use Swift structured concurrency: cancel a previous `Task` and create a new one on each text change; the task sleeps 1.5s then saves.

## Rationale
- The writing views are SwiftUI views without UIKit text view delegates, making `onChange(of:)` the natural hook.
- Combine requires importing Combine + holding `AnyCancellable` sets — more ceremony for a simple debounce.
- Task-based debounce is idiomatic modern Swift (Swift 5.5+), integrates cleanly with `@State`, and expresses the intent clearly: "cancel any pending save, start a new wait period."
- Cancellation semantics are explicit: task is cancelled on view dismissal via `onDisappear`.

## Consequences
- Each writing view holds a `@State private var saveTask: Task<Void, Never>?`.
- `onChange(of: bodyText)` and `onChange(of: titleText)` both reset the save task.
- `onDisappear` cancels the save task and performs a synchronous final save to prevent data loss on abrupt dismissal.
- `Task.sleep` uses `ContinuousClock` for accuracy: `try? await Task.sleep(for: .seconds(1.5))`.
