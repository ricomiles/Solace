# ADR-001: SwiftData over Core Data

Status: Accepted
Date: 2026-05-08

## Context
The app requires local persistence for journal entries and user settings. iOS offers Core Data (mature, Objective-C heritage) and SwiftData (Swift-first, iOS 17+).

## Decision
Use SwiftData with `@Model` classes.

## Rationale
- The brief specifies iOS-native craft and SwiftUI — SwiftData is the natural companion (same WWDC 2023 era as modern SwiftUI patterns).
- SwiftData's `@Model` macro and `@Query` property wrapper eliminate boilerplate NSManagedObject subclasses.
- Brief specifies local-only storage with no sync: SwiftData's full feature set is not needed (no CloudKit, no migration complexity), so its smaller API surface is a benefit.
- iOS 17.0 minimum deployment target (per brief) is satisfied; SwiftData requires iOS 17+.

## Consequences
- Minimum deployment target locked to iOS 17.0. Acceptable per brief.
- Schema migrations for future fields require explicit `SchemaMigrationPlan` — document any future model changes.
- Core Data's NSPredicate is replaced by `#Predicate` macros — slightly less documentation available.
