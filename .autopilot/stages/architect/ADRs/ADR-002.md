# ADR-002: @Observable over ObservableObject

Status: Accepted
Date: 2026-05-08

## Context
`AppState` is a reference-type object that drives cross-screen navigation. SwiftUI offers two observation systems: the legacy `ObservableObject` + `@Published` pattern (iOS 13+) and the new `@Observable` macro (iOS 17+).

## Decision
Use `@Observable` from the Swift Observation framework.

## Rationale
- `@Observable` eliminates the need for `@Published` on each property; all stored properties are automatically observed.
- Views using `@Observable` objects only re-render when properties they actually read change — better than `ObservableObject` which re-renders the whole view tree on any `@Published` change.
- iOS 17+ is already required (SwiftData ADR-001), so there is no compatibility cost.
- `@Environment(AppState.self)` reads more cleanly than `@EnvironmentObject`.

## Consequences
- `AppState` is accessed in views via `@Environment(AppState.self) private var appState`. The `AppState` instance must be injected at the root via `.environment(appState)`.
- Cannot use `@ObservedObject` or `@EnvironmentObject` with `@Observable` types — use `@Environment` or `@Bindable` instead.
