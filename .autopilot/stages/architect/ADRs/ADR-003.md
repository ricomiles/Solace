# ADR-003: NavigationStack + TabView over UIKit navigation

Status: Accepted
Date: 2026-05-08

## Context
The app has 4 peer main sections (Home, Calendar, Search, You) and multiple push/modal screens. Navigation must feel native iOS and support the standard push transition.

## Decision
Use SwiftUI `TabView` for the 4 main sections and `NavigationStack` within each tab for push navigation. Writing views (Mood Check-in, Focused Writing, Prompted Writing) are presented as `.fullScreenCover` modals from the Home tab.

## Rationale
- The brief requires "iOS-native craft" and "native transitions" — SwiftUI's `NavigationStack` delivers the system push animation (right-to-left, 300ms) by default.
- `TabView` with `.tabViewStyle(.automatic)` is the iOS standard for 4 peer sections; maps exactly to the brief's 4 main destinations.
- `.fullScreenCover` for writing views matches the "going into a mode" feel — full screen, dismissible by Done/Close.
- No UIKit navigation controllers needed; UIKit interop is limited to `UIActivityViewController` for export.

## Consequences
- Tab bar is always visible at the bottom of Home, Calendar, Search, You screens. This is not shown in the design mocks (which show individual screens) but is required for navigation.
- The tab bar tint color is set to `terra.200` for the selected item, matching the app palette.
- Writing session state (mood, draft) is managed by `AppState` so it survives tab switching during a session.
